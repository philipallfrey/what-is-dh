Notes

  1. Huma-Num and its associated website https://huma-num.fr provides digital research infrastructure for French-speaking researchers in the social sciences and humanities. See https://documentation.huma-num.fr/humanum-en/. Back to text
  2. In France very large research infrastructures, such as Huma-Num, are denoted IR* (infrastructure de recherche étoile). Back to text
  3. The search engine Isidore (https://isidore.science) harvests social science and humanities data and documents in French, Spanish, and English, and enriches them with standardised metadata. These can be accessed via a keyword search or as Linked Data via a SPARQL endpoint. See https://isidore.science/about. Back to text
  4. The data warehouse Nakala is another service provided by Huma-Num. See https://www.nakala.fr/about. Back to text
  5. Data or documents made publicly available in Nakala are automatically assigned a Document Object Identifier (DOI). See https://documentation.huma-num.fr/nakala/#attribution-dun-doi-a-une-donnee-publique. Back to text
  6. Nakala can be accessed via API if one has a HumanID account, the centralised authentication service for Huma-Num. See https://documentation.huma-num.fr/nakala/#depot-par-lapi. Back to text
  7. When depositing data in Nakala, one can choose to make it public or private. Nakala also supports the Open Archives Initiative Protocol for Metadata Harvesting (OAI-PMH). The logo of Nakala is a koala. See https://documentation.huma-num.fr/nakala/#moissonnage-des-metadonnees. Back to text
